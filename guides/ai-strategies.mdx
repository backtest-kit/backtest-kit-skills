Backtest Kit treats LLM inference as a regular async call inside `getSignal`. Your strategy gathers multi-timeframe candle data, formats it into a message history, sends it to an LLM, and parses the structured JSON response back into a signal DTO. The `Dump` class persists the conversation history and signal records so the model can review past decisions through RAG lookups using the `Memory` class.

## Install the LLM package

```bash
npm install @backtest-kit/ollama backtest-kit agent-swarm-kit
```

## Build a multi-timeframe LLM strategy

Use `getCandles` inside `getSignal` to fetch data at multiple timeframes. Pass the candle arrays to your LLM wrapper alongside any other indicators. The `uuid` from each call links the agent answer and signal record together for later RAG retrieval.

```typescript
import { v4 as uuid } from 'uuid';
import { addStrategySchema, getCandles, Dump } from 'backtest-kit';
import { deepseek } from '@backtest-kit/ollama';

async function myGetSignal(symbol: string) {
  // Fetch candles at four timeframes
  const candles1h  = await getCandles(symbol, '1h', 24);
  const candles15m = await getCandles(symbol, '15m', 48);
  const candles5m  = await getCandles(symbol, '5m', 60);
  const candles1m  = await getCandles(symbol, '1m', 60);

  // Format into message history for the LLM
  const messages = buildMessages(symbol, { candles1h, candles15m, candles5m, candles1m });

  const resultId = uuid();
  const signal = await callLLM(messages); // returns structured JSON

  // Persist the full conversation for RAG retrieval
  Dump.dumpAgentAnswer(messages, {
    dumpId: 'position-context',
    bucketName: 'multi-timeframe-strategy',
    signalId: resultId,
    description: signal.description, // BM25 index keywords
  });

  // Persist the parsed signal record
  Dump.dumpRecord(signal, {
    dumpId: 'position-entry',
    bucketName: 'multi-timeframe-strategy',
    signalId: resultId,
    description: signal.description,
  });

  return { ...signal, id: resultId };
}

addStrategySchema({
  strategyName: 'llm-strategy',
  interval: '5m',
  riskName: 'demo',
  getSignal: deepseek(myGetSignal, 'deepseek-chat', process.env.DEEPSEEK_API_KEY),
});
```

## Persist LLM context with Dump

`Dump.dumpAgentAnswer` writes the full message history to disk and indexes it for BM25 retrieval. `Dump.dumpRecord` writes a flat key-value record (the signal or any structured data). Both are scoped to a `(signalId, bucketName)` pair.

```typescript
import { Dump } from 'backtest-kit';

// Save the agent conversation
Dump.dumpAgentAnswer(messages, {
  dumpId: 'market-analysis',
  bucketName: 'my-strategy',
  signalId: resultId,
  description: 'BTC bullish breakout above 50MA with volume confirmation',
});

// Save the signal record
Dump.dumpRecord(signal, {
  dumpId: 'signal-output',
  bucketName: 'my-strategy',
  signalId: resultId,
  description: signal.note,
});
```

<Tip>
Set `description` to a natural-language summary of the trade rationale. This string is indexed by BM25, so the model can later search for similar historical setups using `Memory.searchMemory`.
</Tip>

## Retrieve past decisions with Memory (RAG)

The `Memory` class provides a BM25-indexed store scoped per `(signalId, bucketName)`. Enable it once at startup, then write and search inside your strategy.

```typescript
import { Memory } from 'backtest-kit';

// Enable once at startup
Memory.enable();

// Inside getSignal — write the current analysis
await Memory.writeMemory({
  memoryId: resultId,
  value: { symbol, signal, timestamp: Date.now() },
  signalId: currentSignalId,
  bucketName: 'trade-history',
  description: 'BTC long entry with R/R 3:1 after bullish engulfing on 1h',
});

// Search for similar past trades before generating a new signal
const similar = await Memory.searchMemory({
  query: 'BTC long entry bullish engulfing',
  signalId: currentSignalId,
  bucketName: 'trade-history',
});

for (const { content, score } of similar) {
  console.log('Relevant history (score:', score, '):', content);
}

// Read a specific record by ID
const record = await Memory.readMemory({
  memoryId: previousSignalId,
  signalId: currentSignalId,
  bucketName: 'trade-history',
});
```

By default `Memory` persists to `./dump/memory/<bucketName>/<signalId>/`. To use an in-memory-only store (faster, no disk I/O), call `Memory.useLocal()` before `Memory.enable()`.

```typescript
Memory.useLocal(); // switch to in-process BM25 only
Memory.enable();
```
