Backtest Kit treats LLM inference as a regular async call inside `getSignal`. Your strategy gathers multi-timeframe candle data, formats it into a message history, sends it to an LLM via `generateObject`, and maps the structured JSON response back to an `ISignalDto`. The `Dump` class persists the conversation history and signal records so the model can review past decisions through RAG lookups using the `Memory` class.

## Install the LLM package

```bash
npm install json-inference openai ollama groq-sdk backtest-kit
```

## Choose a provider

`json-inference` exports `generateObject` and `InferenceName` — one function, one enum, every provider. Change `InferenceName` to switch providers; the schema and messages stay identical.

<CodeGroup>

```typescript deepseek.ts
import { generateObject, InferenceName } from 'json-inference';
import { addStrategySchema } from 'backtest-kit';

addStrategySchema({
  strategyName: 'deepseek-strategy',
  interval: '5m',
  getSignal: async (symbol, when, currentPrice) => {
    return myGetSignal(symbol, currentPrice, InferenceName.DeepseekInference, 'deepseek-chat', process.env.DEEPSEEK_API_KEY);
```

## Signal schema

Define the JSON schema once. `generateObject` validates the response against it and retries automatically on failure.

```typescript
const signalFormat = {
  type: 'object',
  properties: {
    position: {
      type: 'string',
      description: 'Trade direction: "long" (buy) or "short" (sell)',
      enum: ['long', 'short'],
    },
    note: {
      type: 'string',
      description: 'Human-readable description of signal reason',
    },
    price_open: {
      type: 'number',
      description: 'Entry price in USD. Omit to open at current market price',
    },
    take_profit_price: {
      type: 'number',
      description: 'Take profit price in USD. Must be above price_open for long, below for short',
    },
    stop_loss_price: {
      type: 'number',
      description: 'Stop-loss price in USD. Must be below price_open for long, above for short',
    },
    minute_estimated_time: {
      type: 'number',
      description: 'Expected trade duration in minutes before time_expired',
    },
    cost: {
      type: 'number',
      description: 'Entry cost in USD',
    },
    description: {
      type: 'string',
      description: 'User-friendly explanation of risks, min 10 sentences',
    },
    reasoning: {
      type: 'string',
      description: 'Technical analysis, min 15 sentences',
    },
  },
  required: [
    'position',
    'note',
    'take_profit_price',
    'stop_loss_price',
    'minute_estimated_time',
    'cost',
    'description',
    'reasoning',
  ],
} as const;
```

## Build a multi-timeframe LLM strategy

Use `getCandles` inside `getSignal` to fetch data at multiple timeframes. Pass the candle arrays to `generateObject` alongside any other indicators.

```typescript
import { v4 as uuid } from 'uuid';
import { addStrategySchema, getCandles, Dump } from 'backtest-kit';
import { generateObject, InferenceName } from 'json-inference';
import type { ISignalDto } from 'backtest-kit';

async function myGetSignal(
  symbol: string,
  currentPrice: number,
  provider: InferenceName,
  model: string,
  apiKey?: string,
): Promise<ISignalDto | null> {
  // Fetch candles at four timeframes
  const candles1h  = await getCandles(symbol, '1h', 24);
  const candles15m = await getCandles(symbol, '15m', 48);
  const candles5m  = await getCandles(symbol, '5m', 60);
  const candles1m  = await getCandles(symbol, '1m', 60);

  const messages = [
    {
      role: 'system' as const,
      content: [
        `You are a professional trading analyst for ${symbol}.`,
        `Current price: ${currentPrice} USD.`,
        `Analyze the provided candle data and return a structured trade signal.`,
      ].join('\n'),
    },
    {
      role: 'user' as const,
      content: JSON.stringify({
        symbol,
        currentPrice,
        candles1h:  candles1h.slice(-5),
        candles15m: candles15m.slice(-5),
        candles5m:  candles5m.slice(-5),
        candles1m:  candles1m.slice(-5),
      }),
    },
  ];

  const raw = await generateObject(
    provider,
    { format: signalFormat, messages },
    model,
    apiKey,
  );

  if (raw.position !== 'long' && raw.position !== 'short') {
    return null;
  }

  const resultId = uuid();

  // Persist the full conversation for RAG retrieval
  Dump.dumpAgentAnswer(messages, {
    dumpId: 'position-context',
    bucketName: 'multi-timeframe-strategy',
    signalId: resultId,
    description: raw.description,
  });

  // Persist the parsed signal record
  Dump.dumpRecord(raw, {
    dumpId: 'position-entry',
    bucketName: 'multi-timeframe-strategy',
    signalId: resultId,
    description: raw.reasoning,
  });

  return {
    position: raw.position,
    note: raw.note,
    priceOpen: raw.price_open,
    priceTakeProfit: raw.take_profit_price,
    priceStopLoss: raw.stop_loss_price,
    minuteEstimatedTime: raw.minute_estimated_time,
    cost: raw.cost,
  };
}

addStrategySchema({
  strategyName: 'llm-strategy',
  interval: '5m',
  riskName: 'demo',
  getSignal: async (symbol, when, currentPrice) =>
    myGetSignal(symbol, currentPrice, InferenceName.DeepseekInference, 'deepseek-chat', process.env.DEEPSEEK_API_KEY),
});
```

## Persist LLM context with Dump

`Dump.dumpAgentAnswer` writes the full message history to disk and indexes it for BM25 retrieval. `Dump.dumpRecord` writes a flat key-value record (the signal or any structured data). Both are scoped to a `(signalId, bucketName)` pair.

```typescript
import { Dump } from 'backtest-kit';

// Save the agent conversation
Dump.dumpAgentAnswer(messages, {
  dumpId: 'market-analysis',
  bucketName: 'my-strategy',
  signalId: resultId,
  description: 'BTC bullish breakout above 50MA with volume confirmation',
});

// Save the signal record
Dump.dumpRecord(raw, {
  dumpId: 'signal-output',
  bucketName: 'my-strategy',
  signalId: resultId,
  description: raw.reasoning,
});
```

<Tip>
Set `description` to a natural-language summary of the trade rationale. This string is indexed by BM25, so the model can later search for similar historical setups using `Memory.searchMemory`.
</Tip>

## Retrieve past decisions with Memory (RAG)

The `Memory` class provides a BM25-indexed store scoped per `(signalId, bucketName)`. Enable it once at startup, then write and search inside your strategy.

```typescript
import { Memory } from 'backtest-kit';

// Enable once at startup
Memory.enable();

// Inside getSignal — write the current analysis
await Memory.writeMemory({
  memoryId: resultId,
  value: { symbol, raw, timestamp: Date.now() },
  signalId: currentSignalId,
  bucketName: 'trade-history',
  description: 'BTC long entry with R/R 3:1 after bullish engulfing on 1h',
});

// Search for similar past trades before generating a new signal
const similar = await Memory.searchMemory({
  query: 'BTC long entry bullish engulfing',
  signalId: currentSignalId,
  bucketName: 'trade-history',
});

for (const { content, score } of similar) {
  console.log('Relevant history (score:', score, '):', content);
}

// Read a specific record by ID
const record = await Memory.readMemory({
  memoryId: previousSignalId,
  signalId: currentSignalId,
  bucketName: 'trade-history',
});
```

By default `Memory` persists to `./dump/memory/<bucketName>/<signalId>/`. To use an in-memory-only store (faster, no disk I/O), call `Memory.useLocal()` before `Memory.enable()`.

```typescript
Memory.useLocal(); // switch to in-process BM25 only
Memory.enable();
```
